# Generated SaaS input contract

For generated full-stack AI-first SaaS implementation work, proceed only after the selected task, current-intent app-description graph, spec, or backlog supplies or explicitly defers:

- responsible worker, functional agent or explicit internal-only/foundation scope, and execution harness;
- workstream, structured surface id/type/version, and surface action, workstream event, API/MCP/gRPC exposure, workflow/timer/consumer trigger, or other actor adapter when user-facing or externally callable;
- governed tool id, capability id/class, selected Akka substrate, and exposure surfaces;
- `AuthContext`, tenant/customer scope, roles/capabilities, and backend authorization boundary;
- input/output DTOs, redaction, side effects, idempotency, policy/approval/escalation, audit/work traces, runtime evidence target, and required tests.

If these are absent and the work is generated SaaS implementation, route back to `agent-workstream-apps`, `ai-first-saas-worker-decomposition`, and `capability-first-backend`, or block for task-brief repair instead of guessing.

Do not use this contract to block non-SaaS reference-doc work, pure pack maintenance, or tasks that explicitly declare a narrower internal/foundation/docs-only scope.
