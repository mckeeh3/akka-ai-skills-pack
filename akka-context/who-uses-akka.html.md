# Who uses Akka

Akka serves builders, operators, and governance teams. Find the documentation path for your role.

## <a href="about:blank#_builders"></a> Builders

### <a href="about:blank#_application_developers"></a> Application Developers

The broadest group — from senior distributed systems engineers to developers with no distributed systems background. Golden paths and Spec-Driven Development enable consistent delivery regardless of skill level.

- [Spec-Driven Development](sdk/spec-driven-development.html) — The primary way to build with Akka
- [Tutorials](getting-started/index.html) — Hands-on walkthroughs
- [SDK Components](sdk/components/index.html) — Agents, Workflows, Entities, Endpoints, Views, Consumers, Timed Actions
- [Integrations](sdk/integrations/index.html) — Connect to external systems
- [Local Development Setup](sdk/running-locally.html)

### <a href="about:blank#_aiml_engineers"></a> AI/ML Engineers

Focused on agent design, prompt engineering, model selection, and guardrail tuning.

- [Agents](sdk/agents.html) — Build AI-backed agents
- [Guardrails](sdk/agents/guardrails.html) — Runtime policy enforcement
- [Model Provider Configuration](sdk/model-provider-details.html) — Anthropic, OpenAI, Google, AWS Bedrock, and more
- [Agent Memory](sdk/agents/memory.html) — Session and durable memory
- [Multi-Agent Orchestration](sdk/agents/orchestrating.html)

### <a href="about:blank#_product_managers_business_analysts"></a> Product Managers / Business Analysts

Spec-Driven Development means you do not need to write code. Describe requirements in plain language and contribute to system design.

- [Spec-Driven Development](sdk/spec-driven-development.html) — Author specs, not code
