# Support Access Operating Procedure reference

SaaS Owner support access requires explicit tenant-scoped grant, reason, expiry, actor, approver, and audit trail. Support access does not authorize unrelated tenant mutations or cross-tenant browsing.

Confirmed chat tool plan reference: support-access grants, extensions, and revocations are outside the first-pass executable chat catalog. Use review surfaces and approval evidence; do not describe chat as a direct path to create or change support access.
