# Access Review Policy reference

Review stale memberships, dormant admin accounts, excessive roles, support-access grants, last-admin risks, and pending invitations on a regular cadence. Risky remediation should become a decision card with evidence, alternatives, and approver scope.

Confirmed chat tool plan reference: access remediation requests should stay advisory or routed to review surfaces unless they match a backend-owned `human_chat_tool_plan` catalog entry. The proposal must be no-mutation and list confirmation, authorization, idempotency, approval, and trace obligations before any step can execute.
