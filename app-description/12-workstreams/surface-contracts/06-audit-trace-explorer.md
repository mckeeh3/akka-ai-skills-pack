# Surface Contract: Audit Trace Explorer

- surface-id: `surface.audit_trace.search.v1`, `surface.audit_trace.timeline.v1`, `surface.audit_trace.detail.v1`, `surface.audit_trace.investigation_note.v1`
- type/version: list-search + audit-timeline + detail-edit + system-message/v1
- functional agents: Audit/Trace primary; User Admin, Agent Admin, Governance/Policy, and My Account may deep-link into redacted trace evidence.
- payload schema:
  - scoped search filters, result rows, chronological trace detail, authorization basis, policy invocations, prompt/skill/model/tool/data references, redaction labels, export eligibility, and human-authored investigation-note summaries
- allowed actions:
  - search scoped traces → `audit.trace.search`
  - inspect trace detail → `audit.trace.detail`
  - open correlated timeline → `audit.trace.timeline`
  - append investigation note → `audit.trace.investigation_note.append` (annotates workstream trace context only; does not mutate source traces, policy, authorization, or retained evidence)
  - export permitted subset → `audit.trace.export` when enabled; default starter surfaces fail closed or hide export without approval
- UI style notes:
  - render as an enterprise audit trace explorer using the canonical audit timeline/search surface pattern: scoped filters, chronological entries, redaction labels, policy/tool/data evidence, and export eligibility must remain readable and traceable in every named theme
  - use monospace tokens for trace ids and technical references, semantic status labels for denial/redaction/export states, and avoid generic table-only mockups that hide evidence hierarchy
- states:
  - no results, redacted detail, forbidden filter, export denied, note validation error, long search pending, stale live tail if enabled later
- auth/security:
  - all searches and note appends apply tenant/customer filters and caller-specific redaction; support access visibility and sensitive trace reads are audited.
- rendering tests:
  - cross-tenant trace denied, redacted fields marked, prompt/skill/work trace links render, investigation notes record without secret leakage or source-trace mutation, export denial and audit creation verified.
