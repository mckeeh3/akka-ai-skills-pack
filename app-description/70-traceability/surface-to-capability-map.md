# Surface to Capability Map

See `workstream-id-map.md` for canonical app-description surface ids, runtime/API/frontend surface ids, owner functional-agent ids, and dashboard aliases.

This map preserves broad capability grouping while naming the exact governed-tool ids and runtime browser action ids used by the starter workstream implementation. Human-facing executable actions are `browser-tool` exposures; shell navigation/opening actions use the backend-authoritative shell request API and still recheck capability authorization.

## Capability-file anchors

- `secure-tenant-user-foundation` owns user/profile/context, User Admin, notification, invitation, membership, role, support-access, and access-review governed-tools. Canonical User Admin ids such as `admin.users.search`, `admin.invitations.create`, `admin.roles.replace`, and `admin.access_review.read` are mapped above to current runtime `SurfaceAction.governedToolId` values where those differ.
- `frontend-shell-integration-patterns` owns shell request semantics for `show_surface`, `open_workstream`, `refresh_surface`, and `open_attention_item`; the exact shell governed-tool id for opening attention is `attention.open_attention_item`.
- `managed-agent-foundation` owns Agent Admin governed-tools such as `agent_admin.list_definitions`, `agent_admin.get_definition`, `agent_admin.draft_behavior_change`, `agent_admin.submit_behavior_change_for_review`, and `agent_admin.simulate_tool_boundary`.
- `governance-decisions-audit` owns trace and policy governed-tools such as `audit.trace.search`, `audit.trace.detail.read`, `audit.trace.timeline.read`, `audit.trace.investigation_note.append`, `governance.policy.read`, and `governance.policy.simulate`.

## Extension rule

Every new domain surface action must map to a governed capability/tool before implementation. Add the canonical surface id, runtime surface id, `actionId`, capability id, governed-tool id, exposure label, idempotency/approval notes, and denial/audit expectations in this file before adding runtime UI/API behavior.
