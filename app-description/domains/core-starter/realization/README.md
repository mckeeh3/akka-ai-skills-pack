# Core starter realization

Realization files map current-intent nodes to existing backend, API, frontend, test, and validation evidence. They are traceability artifacts only and do not implement runtime behavior.

- [`traceability.md`](traceability.md) captures cross-workstream backbone evidence and explicit gaps.
- Each workstream has its own `realization/akka-components.md`, `realization/api-contracts.md`, and `realization/frontend-routes.md` files.
