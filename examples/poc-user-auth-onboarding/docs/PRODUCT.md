# Product

## Register

product

## Users

All authenticated users of the Akka security SPA: app admins, tenant admins, customer admins, and invited users managing their own account profile. They use the app to confirm access, inspect account status, and perform security administration tasks according to their role.

## Product Purpose

This SPA provides a secure authenticated console for an Akka-backed application using WorkOS. It should make identity, roles, tenants, customers, audit activity, and security settings understandable without slowing down administrative workflows. Success looks like users trusting the access model and quickly finding the next permitted action.

## Brand Personality

Clean, modern, functional. The interface should feel competent, quiet, and deliberately structured, with enough polish to communicate security maturity without looking decorative.

## Anti-references

Avoid neon cyber-security aesthetics, playful consumer styling, generic dark SaaS gloss, and cluttered legacy admin dashboards. Do not let the product become a cold, anonymous enterprise console where every surface looks interchangeable.

## Design Principles

1. Clarity before decoration: every visual choice should help users understand account state, role scope, or next action.
2. Trust through restraint: use consistent structure, visible states, and accessible contrast rather than theatrical security motifs.
3. Role-aware hierarchy: make available sections obvious while keeping unavailable complexity out of the way.
4. Operational calm: the UI should support focused admin work without visual noise or surprise interactions.
5. Template quality: because this is also an integration example, the frontend should read as production-ready and easy to extend.

## Accessibility & Inclusion

Target WCAG AA. Use color-blind safe contrast, visible keyboard focus, 44px minimum touch targets where practical, readable text sizes, semantic structure, and reduced-motion friendly transitions.
