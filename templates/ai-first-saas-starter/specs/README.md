# Starter Specs Seed

This directory is reserved for scaffolded planning artifacts, acceptance checklists, and pending tasks for downstream app extension.

For minimum/starter/basic generated SaaS scopes, the first queued implementation target must be the five core v0 workstream set: My Account, User Admin, Agent Admin, Audit/Trace, and Governance/Policy. Each workstream starts with `markdown_response` v1, workstream log behavior, backend capability boundary, and audit/work trace substrate.

Full-core follow-up tasks must remain visible and explicitly labeled until they are implemented and tested. Do not mark the scaffold as full-core ready just because the five core v0 workstreams are present.
