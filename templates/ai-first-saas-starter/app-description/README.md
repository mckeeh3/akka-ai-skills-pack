# Starter App Description Seed

This directory is reserved for the scaffolded app's maintained description-first source of truth.

Minimum/starter/basic generated SaaS descriptions start with User Admin workstream v0: bootstrap authorization, selected `AuthContext`, bounded `UserAdminAgent`, durable workstream log, `markdown_response`, backend capability boundary, and audit/work trace substrate. Follow-up description work must make the path to full-core readiness explicit instead of treating the minimum starter as complete full core.
