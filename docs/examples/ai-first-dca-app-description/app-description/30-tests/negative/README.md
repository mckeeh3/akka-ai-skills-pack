# Negative Tests

Negative cases define behavior that must be rejected, hidden, or routed to approval instead of being executed.

- `01-security-and-approval-bypass.md` covers:
  - unauthenticated, disabled-user, inactive-membership, wrong-tenant/customer, denied-role/scope, expired-support-access, last-admin, invitation, and frontend secret-boundary cases;
  - supplies tenant isolation, denied capability, unauthorized service/agent access, approval-bypass attempts, and validation failures.
