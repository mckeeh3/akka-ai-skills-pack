# 10 Capabilities

Current files:
- `capabilities-index.md` — capability inventory with the mandatory secure SaaS foundation before DCA-specific automation.
- `01-secure-tenant-user-foundation.md` — first-class foundation capability covering Account, UserProfile, UserSettings, Membership, Role, Permission/Capability, Invitation, AuthContext, `/api/me`, AdminAuditEvent, support access, billing boundary, and tenant/customer isolation.
- `02-lifecycle-orchestration.md` — lightweight lifecycle gate and exception capability contract.
- `03-supplies-autopilot.md` — detailed first vertical capability contract for bounded, policy-gated supply replenishment automation with decision cards, idempotency, audit/trace, exposure surfaces, and tests.
- `04-telemetry-intelligence.md` — lightweight telemetry evidence and source-trust capability contract.
- `05-service-coordination.md` — lightweight service triage, ticket, dispatch, and outcome capability contract.
- `06-meter-billing-review.md` — lightweight meter evidence and billing-impact review capability contract.
- `07-onboarding-installation.md` — lightweight onboarding, installation, and operational-activation capability contract.
- `08-offboarding-retention.md` — lightweight offboarding, deauthorization, retention, and archive capability contract.
- `09-policy-governance.md` — lightweight policy proposal, simulation, activation, and rollback capability contract.
- `10-owner-command-center.md` — lightweight supervision, decision queue, and command-center capability contract.
- `11-audit-outcome-review.md` — lightweight audit, trace, outcome review, retention, and export capability contract.

Purpose: define what the DCA example must support before behavior and implementation mapping. Capabilities are governed backend operations or queries; endpoints, workflows, agents, and entities are exposure or realization choices, not capability roots.
