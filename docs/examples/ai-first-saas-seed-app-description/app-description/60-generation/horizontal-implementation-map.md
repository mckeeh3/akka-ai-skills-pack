# Horizontal Implementation Map

This map is derived from functional agents, surfaces, and governed capabilities. It is not the primary app model.

| Capability | Horizontal Akka/frontend candidates | Notes |
|---|---|---|
| `secure-tenant-user-foundation` | Event Sourced Entities for Invitation/AdminAuditEvent/support-access lifecycle; Key Value Entities for Account/UserProfile/UserSettings/current Role/Permission config where history is not required; Workflows for invitation delivery/acceptance; Views for UserDirectory, Membership, Invitation, AdminAudit, AccessReview; Consumers for email/outbox and audit enrichment; Timed Actions for invite expiry/reminders; HTTP endpoints for `/api/me` and admin APIs; React surfaces for Access/Profile and User Admin. | Preserve tenant/customer scope, backend auth, idempotency, and audit across every surface. |
| `ai-first-work-management` | Event Sourced Entities for Goal/Plan/Decision records; Workflows for long-running plans and approval waits; Agents for planning/summarization/evaluation; Views for mission control and progress; Timed Actions for reminders/rechecks; HTTP/SSE for workstream updates; React Mission Control and Goal Workbench surfaces. | Side-effecting agent tools default to proposal/approval. |
| `governance-decisions-audit` | Event Sourced Entities for Policy, ApprovalRequest, Decision, Audit/Trace records; Workflows for approval gates; Views for decision queue and trace search; Agents for proposal summaries and audit summaries; Consumers for trace enrichment; React decision-card, policy-governance, and audit-trace surfaces. | Human approval remains required for authority expansion and high-impact policy commits. |
| `frontend-shell-integration-patterns` | HTTP endpoints for browser APIs/SSE, static asset hosting for React/Vite build, typed frontend clients, shell components, surface renderers, route/deep-link handlers, frontend tests. | Routes open workstreams/surfaces; they do not define backend authority. |
| `managed-agent-foundation` | Event Sourced Entities for AgentDefinition and governed document/version lifecycle; Key Value Entities for active behavior pointer/cache where suitable; Workflows for proposal/review/activation/rollback; Agents for AgentBehaviorEditorAgent and evaluators; Views for catalogs/history/traces; HTTP endpoints for admin APIs and `readSkill`; React Agent Admin and Governance surfaces. | Prompt/skill content is behavior guidance only; ToolPermissionBoundary and backend AuthContext enforce authority. |
