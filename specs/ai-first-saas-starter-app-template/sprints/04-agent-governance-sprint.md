# Sprint 04: Agent Governance and Runtime Foundation

## Goal

Complete the governed runtime agent foundation in the starter app: seed documents, AgentDefinition, prompts, skills, manifests, tool boundaries, prompt assembly, readSkill authorization, traces, and Agent Admin/governance surfaces.

## Outcomes

- Implementation-developed default agent behavior is imported idempotently into governed storage.
- Agent Admin and Governance/Policy capabilities are real backend capabilities with workstream UI surfaces.
- Prompt/skill/manifest/tool-boundary edits use proposal/review/activation semantics.
- Agent work, prompt assembly, skill load, and tool-boundary decisions emit traces/audit.
- Tests prove unauthorized behavior changes, unassigned skill reads, disabled agents, and authority expansion are denied.
