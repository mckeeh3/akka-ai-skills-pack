# Sprint 01: Template Scope and Inventory

## Goal

Make the starter-app target explicit before implementation: scope, acceptance criteria, canonical source/template location, install modes, and disposition of old DCA/seed assets.

## Outcomes

- Starter app purpose and full-core scope are documented.
- Canonical template location and package/export strategy are chosen.
- Existing executable/reference assets are inventoried as reuse, migrate, quarantine, archive, or delete-later.
- The first implementation task can proceed without guessing where code belongs.
