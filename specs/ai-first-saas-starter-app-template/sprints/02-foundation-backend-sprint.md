# Sprint 02: Secure Foundation Backend

## Goal

Create the runnable backend foundation for the starter app: identity seams, local authorization, tenancy, invitation onboarding, user administration, `/api/me`, authorization, audit, and tenant-isolation/security tests.

## Outcomes

- Starter backend has a coherent Java package and project layout.
- Full-core foundation APIs and components are real, not only docs or fixtures.
- Invitation, membership, role/capability, profile/settings, support-access boundary, and audit flows are demonstrable.
- Backend tests cover success, validation, idempotency, forbidden access, disabled users, role denial, tenant isolation, and audit.
