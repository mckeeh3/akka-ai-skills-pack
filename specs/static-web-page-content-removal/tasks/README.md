# Static Web Page Content Removal Task Briefs

Use these task briefs with `../pending-tasks.md`.

Rules:

- Execute one task per fresh harness context.
- Read only the selected task's required files plus any files discovered by its searches.
- Mark the selected task `in-progress` before editing.
- Do not work on another task in the same session.
- Run the required checks.
- Make exactly one git commit for the selected task before marking it `done` when feasible.
- Include the queue-status update in the same commit as the task changes.
- If a commit hash cannot be recorded without amending the same commit, record the commit message in the task notes instead.
