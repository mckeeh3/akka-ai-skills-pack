# Sprint 2: Description and Decomposition Integration

## Sprint goal

Make capability modeling a first-class output of app-description maintenance, PRD/spec intake, and Akka solution decomposition.

## Scope

- Update app-description guidance so maintained app descriptions can capture capability inventories and capability readiness.
- Update decomposition guidance so broad product input yields capability models before component selection.
- Update PRD/spec/backlog planning guidance so generated tasks preserve capability authority, scope, audit, and exposure decisions.
- Define readiness criteria for capability-first generation.

## Expected outputs

- Revised app-description skills/docs for capability inventory and capability changes.
- Revised `akka-solution-decomposition` guidance.
- Revised PRD/spec/backlog intake guidance.
- Example app-description capability inventory, if small and useful.

## Acceptance behavior

A high-level feature request should produce capability candidates with schemas, auth/scope, side effects, audit, approval rules, and exposure surfaces before the agent chooses entities, workflows, views, endpoints, or tools.

## Defer list

- Broad component implementation rewrites.
- Executable reference examples.
