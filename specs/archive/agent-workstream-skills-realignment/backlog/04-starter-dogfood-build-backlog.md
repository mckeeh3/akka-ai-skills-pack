# Backlog 04: Starter Dogfood and Queue Realignment

## Outcome

The starter app implementation plan becomes a dogfood proof of the realigned skills pack: work starts from functional agents and structured surfaces, maps actions to governed capabilities, then implements Akka/frontend/test assets end to end.

## Backlog items

1. **Audit current starter Sprint 07 queue**
   - Identify tasks that remain vague, slice-only, or component-first.
   - Map each current task to functional agents, surfaces, and capabilities or mark it for supersession.

2. **Supersede vague starter tasks**
   - Replace broad durable-slice tasks with vertical workstream/surface/capability tasks.
   - Preserve useful validation and acceptance tasks.

3. **Add full-core vertical starter tasks**
   - Add tasks for Access/Profile, User Admin, Invitation Onboarding, Agent Admin, Audit/Trace, Governance/Policy, and shell/realtime/static hosting.
   - Each task must be self-contained for a fresh harness session and must commit on completion.

4. **Sprint review and completion decision**
   - Run the workstream design review checklist against the updated skills and starter queue.
   - Either declare skills realignment complete enough for PRD-driven implementation or add another sprint with concrete remaining refinements.
