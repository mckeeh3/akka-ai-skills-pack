# Sprint 04: Runtime Smoke and Verification

## Objective

Prove or explicitly block/defer full-core SaaS readiness through local Akka/API/UI validation after the core foundation, user-admin, managed-agent, audit, and governance slices are implemented.

## Scope

- End-to-end runtime smoke checklist.
- Backend tests, frontend tests/typecheck/build, static secret-boundary checks, and focused manual smoke notes.
- Completion verification against `README.md` done state and app-description readiness docs.

## Acceptance criteria

- Full-core readiness is claimed only when real local runtime evidence supports it.
- If gaps remain, they are appended as bounded follow-up tasks before a new terminal verification task.
- App-description readiness docs are updated to match the verified state.
