# Sprint 4: Core App PRD and Seed Realignment

## Goal

Replace weak getting-started behavior with a canonical core app PRD that requires foundation functional agents and demonstrable workstream behavior.

## Scope

- Create or revise the core app PRD so the minimum core app includes the agent workstream shell and foundation functional agents.
- Define required foundation agents: Access/Profile, User Admin, Agent Admin, Audit/Trace, Governance/Policy, and optional Billing/Subscription where relevant.
- Define module/sprint progression around adding functional agents and surfaces.
- Update getting-started prompt/docs to launch from the canonical PRD and require scope selection.
- Update starter core app-description references to match workstream architecture.

## Out of scope

- Implementing the core app in source code.
- Expanding every ERP/CRM example beyond representative functional-agent examples.

## Done criteria

- A generated “core app” cannot be considered complete if User Admin and Agent Admin functional agents are missing, unless the user explicitly selected a narrower Module 1-only scope.
- Getting-started flow routes through the canonical PRD and readiness gates.
