# Sprint 2: Description Model Realignment

## Goal

Realign app-description guidance so generated apps are specified as functional agents, internal agents, surfaces, capabilities, and horizontal implementation maps.

## Scope

- Revise app-description architecture docs and bootstrap guidance.
- Add or revise app-description skills for functional-agent and surface modeling.
- Ensure capability inventory is derived from functional agents and surfaces.
- Update readiness checks so an app is not generation-ready if consequential authenticated work areas are not agent-backed.
- Update example app-description trees to demonstrate the new model.

## Out of scope

- Full implementation generation.
- Broad UI component code examples except references needed by examples.

## Done criteria

- New app descriptions start from role-authorized functional agents and their workstream surfaces.
- Horizontal Akka component maps are derived artifacts, not the starting point.
- Existing examples no longer teach page hierarchy as the primary generated-app model.
