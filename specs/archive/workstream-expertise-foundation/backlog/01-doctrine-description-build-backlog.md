# Backlog 01: Doctrine and Description Ownership

## Outcome

The pack has a canonical doctrine for workstream expertise and app-description guidance has an authoritative place to model it.

## Backlog items

1. **Create workstream expertise doctrine**
   - Define workstream expert bundle fields.
   - Explain skill vs reference document vs capability vs tool boundary.
   - Define how expertise fits under functional agents, surfaces, and capabilities.

2. **Align app-description architecture and functional-agent modeling**
   - Add or update artifact ownership for `12-workstreams/workstream-expertise/**`.
   - Require each functional agent to list expert skills, reference docs, manifests, tool boundaries, capabilities, traces, and tests.

3. **Align readiness and traceability guidance**
   - Ensure readiness review blocks functional-agent completion if expertise artifacts are missing or vague.
   - Ensure traceability maps connect functional agents to expertise docs, capabilities, surfaces, tests, and observability.

4. **Sprint review and next-task validation**
   - Review the doctrine/description model for ambiguity.
   - Decide whether Sprint 02 should implement separate reference-document governance or a constrained first pass using skill governance.
