# Runtime Expertise Gap Matrix

## Purpose

This audit identifies governed-runtime gaps that must be closed before the pack can claim full workstream expertise loading support. It does not implement the fixes.

Scope follows `TASK-WEF-02-001`: skill governance, reference documents, expertise manifests, loaders, seed import, tool boundaries, traces, and tests.

## Summary verdict

Current guidance and examples strongly cover governed procedural skills: `SkillDocument`, `SkillVersion`, per-agent `AgentSkillManifest`, compact prompt assembly, `readSkill(skillId)`, `ToolPermissionBoundary` checks, `SkillLoadTrace`, seeded default prompts/skills/manifests/boundaries, and several runtime tests.

The main gap is that **workstream expert bundles and reference documents are not yet first-class runtime concepts**. Existing runtime guidance can be extended, but today it still centers on skill manifests and `readSkill`; it does not fully define reference ids, reference manifests, `readReferenceDoc`, reference-load traces, seed packaging, denied reference loads, or bundle-level readiness tests.

Recommended Sprint 02 direction: add explicit governed reference-document loading guidance and align runtime/seed/tool-boundary/testing docs around a compact **expertise manifest** containing separate skill and reference entries. Prefer first-class reference-document semantics. If an interim representation uses `SkillDocument` with `documentKind: reference`, preserve the same manifest, loader, trace, authorization, and test distinctions.

## Minimum closure criteria for Sprint 02 runtime guidance

Sprint 02 runtime guidance should be considered sufficient for seed/example work only when it explicitly defines:

1. Workstream expertise manifest structure with separate skill and reference sections.
2. Stable reference ids and either first-class `ReferenceDocument`/`ReferenceVersion`/`AgentReferenceManifest` records or a constrained interim governed-document representation.
3. Authorized reference loader tool contract and tool-boundary grant category.
4. Allowed and denied reference-load trace records linked to `AgentWorkTrace`.
5. Seed bundle fields for references, reference manifests, and bundle cross-references.
6. Prompt assembly updates that include compact expertise entries only, not full skill/reference bodies.
7. Tests for allowed loads, denied unassigned/cross-tenant/inactive/oversized loads, boundary denials, seed import, trace emission, and no authority expansion from expertise text.
