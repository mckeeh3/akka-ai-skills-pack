# Sprint 05: Verification Loop

## Goal

Verify the expanded chat tool catalog through the intended local runtime/API/UI path at the stated scope.

## Required verification behavior

Terminal verification must:

1. compare completed work against the README done state;
2. verify the catalog inventory/classification covers all foundation workstream surface actions;
3. verify expanded executable/proposal/approval-gated paths by representative API/UI tests;
4. verify blocked/surface-only classifications are safe and documented;
5. verify no pre-confirmation mutation, exact confirmation, idempotency, partial failure/recovery, provider fail-closed behavior, and traces;
6. run required backend/frontend validators/checks;
7. append follow-up tasks plus a new terminal verification task when gaps remain.

## Completion signal

The mini-project closes only when terminal verification records no material coverage or safety gaps for the expanded first scope.
