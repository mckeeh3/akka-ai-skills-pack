# Sprint 05: Trim and Repeat Review

## Objective

Run repeated whole-pack review passes for stale active content, remove/rewrite remaining tech debt, and verify installable-pack consistency.

## Scope

- Whole `skills/` and `docs/` active guidance.
- `pack/manifest.yaml` and packaging/resource references when touched docs/skills are removed or renamed.
- Existing related specs only for recommendations or follow-up mini-projects, not wholesale migration.

## Ordered work areas

1. Run stale-term searches and inspect hits in active guidance.
2. Remove or rewrite duplicate/stale active docs identified by previous sprints.
3. Confirm manifest and resource references do not point to removed content.
4. Produce final alignment review and append follow-up tasks if gaps remain.

## Acceptance criteria

- Final review records search terms, inspected hit classes, and disposition.
- Active intake/planning guidance aligns with current goals.
- Any remaining legacy content is explicitly mechanics-only, non-generated-app, or outside active guidance.
- Terminal verification can decide whether the mini-project is complete or append another task group.
