# Sprint 01: Inventory and Archive Plan

## Objective

Create a precise inventory of existing user-intent processing skills/docs and define how the pre-intent-compiler material will be archived, retained, replaced, or lightly realigned.

## Scope

- Identify active skills/docs that process user intent input.
- Classify each artifact as archive, replace, keep active with reference update, or untouched.
- Decide safe archive mechanics for docs and skills without breaking install validation.
- Produce an archive/replacement matrix.

## Acceptance criteria

- Inventory covers the intent-processing skill/doc families named in `conversation-capture.md`.
- Archive strategy distinguishes docs from skills and accounts for installer/reference validation.
- No active artifact is moved until the migration plan is explicit.
