# Full-Core Realization Map

## Purpose

This map turns the canonical core app PRD into a downstream implementation contract for the skills pack. It preserves the required full-core scope, maps it to app-description layers and capability-first backend contracts, and names the Akka/frontend substrates and tests future tasks must specify or implement.

Primary input: `docs/examples/core-ai-first-saas-input/10-canonical-core-app-prd.md`.

## Scope gates

| Scope | Required meaning | Generation label |
|---|---|---|
| Full core | Includes Access/Profile, User Admin, Agent Admin, Audit/Trace, and Governance/Policy functional agents; includes invitation onboarding, full user admin, governed runtime agent records, prompt/skill/manifest/tool-boundary governance, workstream UI, audit/trace, and security tests. | `full core` |
| Module 1-only | Includes WorkOS/AuthKit sign-in, `/api/me`, selected AuthContext, minimal profile/context display, authenticated shell, and Access/Profile only. Defers User Admin, Agent Admin, invitation lifecycle, governed prompts/skills/manifests/tool boundaries, unified audit/work trace UI, and governance loops. | `Module 1-only / not full core` |

Full-core generation is **not ready** if User Admin or Agent Admin are missing or deferred. Defers are allowed only when the selected scope is explicitly Module 1-only or another named narrower module scope.

## Cross-cutting readiness checks

A generated plan, app-description, or code task fails full-core readiness when any of these are absent without an explicit narrower-scope label:

1. Access/Profile, User Admin, Agent Admin, Audit/Trace, and Governance/Policy functional agents.
2. Complete invitation lifecycle with Resend production boundary and local/dev/test captured outbox.
3. Full user administration: discover/list/search users, membership lifecycle, role/capability assignment, disabled access, access review, support-access, last-admin protection, and admin audit.
4. Governed runtime agent records: AgentDefinition, PromptDocument/PromptVersion, SkillDocument/SkillVersion, AgentSkillManifest, ToolPermissionBoundary, ModelConfigurationRef, seed import, readSkill, PromptAssemblyTrace, SkillLoadTrace, AgentWorkTrace.
5. Workstream UI shell with backend-derived role-authorized functional rail, main workstream, persistent composer, structured surfaces, capability actions, error/forbidden/stale states, and accessibility/responsive coverage.
6. Backend authorization for every protected endpoint, component command, view query, stream, workflow action, consumer side effect, timer action, agent tool, and internal-agent operation.
7. Tenant/customer isolation, disabled-user, role/capability denial, audit/trace creation, redaction, idempotency, approval, tool-boundary, and frontend secret-boundary tests.

## Follow-up task contract

Future tasks should use this map as the source of truth for full-core scope. When a task narrows to one module or slice, it must copy the relevant capability ids, AuthContext/scope, Akka substrate choices, exposure surfaces, and test obligations into that slice rather than re-deciding the architecture.
