# Sprint 01: Dogfood Release Readiness

## Objective

Turn positive manual dogfood feedback into release-readiness evidence and close any bounded validation gaps for the attention-backed starter scope.

## Scope

- Record manual observation evidence.
- Run fresh scaffold automated checks.
- Validate visible attention behavior and edge cases at v1/v2 scope.
- Review docs/handoff and future-work boundaries.
- Verify completion or append blockers/fixes.

## Acceptance criteria

- Fresh scaffold works and targeted backend/frontend checks pass.
- Dogfood evidence covers left rail, My Account, dashboards/surfaces, and producer-driven updates.
- Security/edge checks cover redaction, denied access, lifecycle, provider/fail-closed, and frontend-only authority guardrails.
- Docs/handoff do not overclaim v3/future behavior.
