# Sprint 05: Packaging and Install Modes

## Goal

Package the starter app as a skills-pack resource/template and add explicit scaffold/init behavior without disrupting skills-only installs.

## Outcomes

- Pack includes the starter template in a documented resource location.
- Install/init scripts or docs expose skills-only and starter-app scaffold modes.
- Existing project/global installs do not overwrite user application code.
- Starter scaffold records selected Java base package and can be built/tested after copy.
- User/developer guides describe extending the starter app as the preferred new-project path.
