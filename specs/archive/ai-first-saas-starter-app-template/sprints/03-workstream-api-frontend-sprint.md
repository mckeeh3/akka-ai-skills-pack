# Sprint 03: Workstream API and Frontend Wiring

## Goal

Wire the canonical React/Vite/TypeScript workstream UI to real starter backend APIs while preserving fixture clients as tests/dev references only.

## Outcomes

- `/api/me`, User Admin, Access/Profile, Audit/Trace, Governance/Policy, and Agent Admin API contracts are implemented or stubbed with backend-authoritative denial/audit behavior.
- Frontend API clients call same-origin real endpoints in production mode.
- Workstream shell, rail, composer, surfaces, actions, deep links, and realtime/stale behavior remain canonical.
- Frontend and endpoint tests prove the UI is packaged and wired to real APIs.
