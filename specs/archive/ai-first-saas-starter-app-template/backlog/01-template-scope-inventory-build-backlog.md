# Build Backlog 01: Template Scope and Inventory

## Items

1. Define starter app scope, acceptance criteria, and explicit non-goals.
2. Inventory existing DCA, seed, frontend fixture, and static resource examples for reuse/migration/quarantine/archive decisions.
3. Choose canonical template source and installed-pack resource locations.
4. Define initial extension workflow: scaffold starter → update app-description/specs → extend capabilities/UI/tests.

## Completion signal

Future implementation sessions know what to build, where it belongs, and which old assets are no longer canonical.
