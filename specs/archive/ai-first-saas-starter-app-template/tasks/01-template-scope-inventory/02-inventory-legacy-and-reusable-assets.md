# TASK-STARTER-01-002: Inventory legacy and reusable assets

## Purpose

Classify existing DCA, seed, frontend, and static-resource assets so the starter app can reuse useful pieces without letting stale examples remain canonical.

## Required reads

- `specs/ai-first-saas-starter-app-template/starter-app-scope-and-acceptance.md`
- `docs/ai-first-examples-and-tests-gap-list.md`
- `docs/web-ui-pattern-selection.md`
- `docs/workstream-ui-reference-architecture.md`
- `frontend/src/workstream/**`
- `src/main/java/com/example/**`
- `src/main/resources/static-resources/**`
- `docs/examples/**`

## Expected outputs

- `specs/ai-first-saas-starter-app-template/legacy-and-reusable-asset-inventory.md`

## Done criteria

- Existing major executable/reference assets are classified as reuse, migrate, mechanics-only, quarantine, archive, or delete-later.
- The inventory identifies which assets must not be referenced as canonical full-core guidance after replacement.
- Queue status is updated and changes are committed.
