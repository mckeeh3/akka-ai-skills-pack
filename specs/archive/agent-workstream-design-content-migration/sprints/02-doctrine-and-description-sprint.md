# Sprint 2: Doctrine and App-Description Alignment

## Goal

Align core doctrine and app-description guidance around the canonical functional-agent workstream model.

## Scope

- Update doctrine docs that define generated-app architecture.
- Update app-description architecture and bootstrap guidance.
- Standardize `55-ui/` file-set guidance.
- Clarify that `12-workstreams/` owns the application model and `55-ui/` owns browser rendering/interaction semantics.
- Normalize functional/context-area agent terminology in doctrine and description-first guidance.

## Out of scope

- Focused implementation skills not directly involved in app-description or design doctrine.
- Example migration, except for links needed by doctrine.

## Done criteria

- Core docs and description-first skills no longer contain conflicting UI/application structure guidance.
- New app descriptions are directed to functional agents, workstreams, and structured surfaces before browser routes or pages.
