# Sprint 3: Skill Routing and Generation Alignment

## Goal

Tighten skills that route UI, surface, generation, readiness, and web-UI implementation work so they consistently preserve functional-agent workstream design.

## Scope

- Update `ai-first-saas-ui-surfaces` to require functional-agent/workstream placement for every selected surface.
- Update web UI routing skills and docs to route generated SaaS UI to `frontend/src/workstream/**`.
- Strengthen legacy page/screen/static example quarantine notes.
- Verify readiness and generation gates block page-first or CRUD-first generated SaaS designs.

## Out of scope

- Editing all component implementation skills unless they directly contradict the workstream model.
- Frontend code changes.

## Done criteria

- UI/surface skills select and realize surfaces inside functional-agent workstreams.
- Generation and web UI skills point to the canonical workstream reference and do not promote legacy screen/page examples.
