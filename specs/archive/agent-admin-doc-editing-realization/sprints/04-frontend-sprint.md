# Sprint 04: Frontend realization

## Goal

Revise the React/Vite workstream UI so Agent Admin renders the new doc-editing surfaces and no longer centers stale governance-console flows.

## Tasks

- `AADE-04-001`: frontend contracts, API types, and fixtures for Agent Admin doc editing.
- `AADE-04-002`: frontend browsing/doc/version surfaces.
- `AADE-04-003`: frontend editing/create/delete/trace flows and stale governance UI cleanup.

## Acceptance

- Frontend supports blank state, optional dashboard, agent list, agent detail, prompt doc, skill doc, reference doc, edit session, version history, adjacent diff, restore, skill/reference create/delete, and runtime traces.
- Edit input appears only on current/latest versions.
- Historical versions show read-only state.
- Contract tests reflect current Agent Admin intent.
