# Goals and Objectives

- durable goal object:
  - fields: goal id, tenant id, owner, objective, constraints, success criteria, priority, target date, status, outcome links
  - required behavior: every delegated work item starts from a durable goal or an explicitly linked task
- supported objective classes:
  - explore or draft a plan
  - execute bounded operational work
  - review a recommendation
  - improve a policy or example set
  - inspect audit/outcome history
- retained human authority:
  - humans define objectives and success criteria
  - humans approve high-impact or policy-bound actions
  - humans commit policy changes that expand authority
- success criteria:
  - goal progress is inspectable
  - execution plan is auditable before or during activation
  - outcomes can be linked back to goals, decisions, and agent work
