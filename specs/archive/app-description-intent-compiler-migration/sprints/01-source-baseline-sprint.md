# Sprint 01: Temporary Source Baseline and Implementation Inventory

## Goal

Create safe migration inputs without treating legacy descriptions as ongoing authority.

## Tasks

- Archive the current `app-description/` as temporary migration input.
- Inventory root implementation, frontend, tests, docs, and active specs that describe actual starter-app behavior.
- Classify findings into reusable foundation references, core starter current intent, stale/legacy content, and unresolved drift.

## Done criteria

- A temporary archive exists for migration tasks.
- A source inventory identifies legacy-description and implementation evidence for the current-intent reconstruction.
- No active root current-intent content has yet been replaced without a recovery path.
