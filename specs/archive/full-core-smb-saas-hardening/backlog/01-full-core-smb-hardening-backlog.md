# Backlog: Full-Core SMB SaaS Hardening

## Goal

Create a disciplined program plan for making the five-core starter functionally complete for SMB SaaS use while improving the skills pack through executable implementation.

## Suggested harness task breakdown

1. Create this umbrella planning scaffold.
2. Define the SMB full-core baseline and visual/UX standards.
3. Produce a workstream-by-workstream capability/surface/agent outline.
4. Create the first wave of child mini-projects and queues.
5. Verify umbrella readiness and record next child project to execute.

## Required checks

- `git diff --check`
- targeted `rg` checks proving baseline, workstreams, surfaces, request/response agents, AutonomousAgent/internal workers, deterministic services, and visual standards are represented

## Acceptance criteria

- The program has a clear path from v0 to SMB-complete core.
- Child mini-projects are bounded and sequenced.
- Implementation begins with the highest-leverage baseline/wave tasks, not with ad hoc code changes.
