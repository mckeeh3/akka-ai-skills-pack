# Sprint 01: Release Readiness Validation

## Objective

Verify the repository is ready for a new release after five-core v0 completion and core PRD reconciliation.

## Scope

- starter fullstack validation;
- version consistency;
- pack build;
- source install and scaffold checks;
- release-facing docs review;
- release-readiness handoff.

## Acceptance criteria

- Required commands have explicit pass/fail/skip evidence.
- Any release blocker is converted into a bounded follow-up task before terminal verification.
- No generated `dist/` artifacts are committed unless explicitly required by release policy.
