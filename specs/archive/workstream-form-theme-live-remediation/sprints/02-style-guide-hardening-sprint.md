# Sprint 02: Style Guide Hardening

## Objective

Update style guidance so future generated workstream forms cannot be accepted with browser-default controls or delayed theme application.

## Scope

- `docs/web-ui-style-guide.md`.
- Focused web UI skills/checklists only if needed by search.

## Acceptance criteria

- Canonical style guide explicitly requires styled structured-surface form controls.
- Named-theme selection guidance requires immediate local preview on selection and durable persistence through the governed save path.
- Guidance rejects native/default-looking form controls in generated workstream surfaces.
