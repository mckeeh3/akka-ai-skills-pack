# Sprint 01: Core doctrine and routing

## Goal

Make governed runtime agent behavior management part of the mandatory secure AI-first SaaS foundation in repository doctrine and skill routing.

## Scope

This sprint updates source-of-truth guidance so a new core app bootstrap includes:

- tenant-scoped `AgentDefinition` and behavior profile management;
- governed `PromptDocument` / `PromptVersion`;
- governed `SkillDocument` / `SkillVersion`;
- per-agent `AgentSkillManifest`;
- tool permission boundaries;
- prompt assembly, skill loading, and work traces;
- behavior editing agents for prompt/skill/manifest/tool-boundary changes;
- UI surfaces for agent catalog/detail, prompt governance, skill governance, manifests, tools, editing-agent proposals, and traces.

## Acceptance

- Core doctrine no longer implies these are optional advanced add-ons for generated AI-first SaaS foundations.
- Routing map points foundation/new-app flows through the existing agent behavior profile, governed document, prompt governance, skill governance, and work trace skills.
- Admin-agent guidance allows one governed skilled agent or multiple specialized agents.
