# Backlog 03: Planning and task generation

## Purpose

Make planning skills produce concrete, fresh-session-sized implementation tasks for the governed runtime agent foundation.

## Delivery goal

PRD/spec/backlog flows create explicit foundation tasks for managed agents before app-specific domain features.

## Capability contracts

- `plan.agent-foundation.decompose`: include governed runtime agent capabilities in solution plans.
- `plan.agent-foundation.materialize-tasks`: create bounded tasks for each component/UI/test family without collapsing the work.

## Suggested harness task breakdown

1. Update solution decomposition and PRD-to-specs/backlog guidance.
2. Update backlog-to-pending-tasks, slice-to-backlog, and task-brief guidance.

## Done criteria

- Planning outputs include concrete tasks for AgentDefinition, PromptDocument/PromptVersion, SkillDocument/SkillVersion, AgentSkillManifest, `readSkill`, traces, behavior editing agents, tool boundaries, UI, and tests.
- Pending task guidance requires these tasks to be separate and foundation-first.
