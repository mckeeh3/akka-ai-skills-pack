# Sprint 01: Architecture Decision and Inventory

## Objective

Turn the discussed fork-and-extend direction into an explicit repository architecture decision, path map, and migration inventory before moving files.

## Scope

- Define the target root app layout and `skills-pack/` layout.
- Inventory app, template, skills-pack, docs, tools, examples, and active specs that need relocation or reference updates.
- Identify extension boundaries for domain-specific work.

## Acceptance criteria

- A documented target layout exists.
- Each major existing root/template/skills-pack asset is classified as promote, move, remove, archive, retain, or update-reference.
- A path compatibility map exists for future tasks.
- No source files are moved in this sprint except planning artifacts.
