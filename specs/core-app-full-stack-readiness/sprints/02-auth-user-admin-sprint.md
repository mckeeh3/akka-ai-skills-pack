# Sprint 02: Auth, Invite Onboarding, and User Administration

## Goal

Close realization gaps for the secure SaaS identity/admin foundation: WorkOS/AuthKit, `/api/me`, invite onboarding, memberships, roles, support access, scoped admin views, and User Admin workstream surfaces.

## Outcomes

- The pack has implementation-ready contracts and reference assets for full user onboarding via invites.
- SaaS Owner, Tenant, and Customer admin boundaries are concrete.
- User Admin surfaces and APIs can be generated with tenant isolation and audit tests.
