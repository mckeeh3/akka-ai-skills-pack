# AI-first SaaS user administration remediation

This planning package closes remaining gaps in the skills pack's secure AI-first SaaS foundation.

Non-negotiable direction:
- Initial generated apps must be fully capable of email-invite user onboarding.
- Admin users must be fully capable of managing users, memberships, roles/scopes, invitations, access state, and audit visibility within their authority boundary.
- AI agents should offload as much routine administration work as safely possible: drafting invites, recommending roles, detecting stale access, identifying risky changes, summarizing audit context, and creating decision cards for high-risk admin actions.
- Human admins remain supervisors, approvers, policy owners, auditors, and exception decision makers.
- Security and user-management behavior must be foundation behavior, not later polish.

Execution rule:
- Execute exactly one task per fresh harness session.
- Each task must commit its work to git before completion.
- Do not combine tasks.
