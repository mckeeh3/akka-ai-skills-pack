# Sprint 04: Verification and Queue Repair

## Goal

Verify the completed task group against the mini-project done state and append follow-up tasks if material gaps remain.

## Scope

- Compare implemented app-description, backend/API, frontend, tests, and evidence against `README.md` done state.
- Run appropriate validation checks for touched areas.
- Record completion evidence or append bounded follow-up tasks plus a new terminal verification task.

## Acceptance

- Verification states whether the current task group is complete.
- Verification states whether the overall mini-project is complete.
- Remaining material gaps are not silently accepted; they become new bounded pending tasks or explicitly deferred non-goals.
