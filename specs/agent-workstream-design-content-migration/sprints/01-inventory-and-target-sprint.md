# Sprint 1: Inventory and Target

## Goal

Inventory design-content drift around functional/context-area agent workstreams, then define the canonical terminology, file-set, ownership, and review-checklist targets before broad edits begin.

## Scope

- Identify drift across docs, skills, examples, and generation/readiness guidance.
- Define the canonical `12-workstreams/` vs `55-ui/` ownership boundary.
- Define the canonical `55-ui/` file set for generated full-stack AI-first SaaS apps.
- Define terminology rules for functional/context-area agents.
- Create the compact design review checklist target.

## Out of scope

- Rewriting all docs and skills.
- Migrating examples.
- Changing frontend implementation code.

## Done criteria

- A drift inventory exists.
- The target content rules are documented.
- Follow-on tasks know exactly which docs, skills, and examples to edit.
