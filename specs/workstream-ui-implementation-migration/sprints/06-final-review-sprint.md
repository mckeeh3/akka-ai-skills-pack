# Sprint 6: Final Consistency Review

## Goal

Verify the repository no longer contains stale frontend reference drift that undermines workstream UI generation.

## Scope

- Search for page-first, screen-first, supervisory-console, or chatbot-bolt-on implementation examples that should be revised, archived, or explicitly marked legacy.
- Verify `frontend/` is the canonical reusable workstream UI reference.
- Verify generated/static resources are either current build outputs or clearly marked legacy fixtures.
- Verify skills, docs, app-description examples, frontend source, and tests point in the same direction.
- Record migration completion summary and any follow-up backlog.

## Out of scope

- New feature implementation after the reference is coherent.

## Done criteria

- The migration has a completion summary.
- Remaining drift is either removed, revised, or tracked as follow-up work.
