# Sprint 02: Seed Surface Style Review

## Objective

Review starter core app-description surface contracts for consistency with the updated core domain surface style guidance and patch small gaps if found.

## Scope

- Starter core app-description `12-workstreams/surface-contracts/**`.
- Starter core app-description `55-ui/style-guide.md` as reference.
- Small documentation-only patches to seed surface contracts when they lack named-theme/style alignment or contradict the core domain docs.

## Acceptance criteria

- Seed surface contracts remain consistent with canonical style guide and core domain surface docs.
- No active seed surface contract implies obsolete style ids, old orange/coral defaults, mode-first theme selection, or generic mockup-only surfaces.
- Any larger gaps are queued as follow-up instead of being silently expanded.
