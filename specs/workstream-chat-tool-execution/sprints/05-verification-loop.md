# Sprint 05: Verification Loop

## Goal

Verify the root app mini-project done state through the intended local Akka/API/UI path at the stated scope, or append follow-up tasks when gaps remain.

## Verification requirements

The terminal verification task must:

1. compare completed work against the README done state;
2. verify deterministic surface routing still works before plan proposal;
3. verify the User Admin example does not mutate before confirmation and executes after confirmation when authorized;
4. verify all five foundation workstreams have representative confirmed chat tool-plan coverage;
5. verify provider/model missing configuration fails closed rather than faking planning success;
6. verify audit/work traces and result/partial-failure surfaces are present;
7. run required backend/frontend checks and a local API/UI/manual smoke path when available;
8. append bounded follow-up tasks plus a new terminal verification task when material gaps remain.

## Completion signal

The mini-project is complete only when terminal verification records that the README done state is achieved or explicitly scopes any remaining gaps into newly appended follow-up tasks.
