# Sprint 01: User Admin Access Management Slice

## Objective

Make User Admin usable for SMB member status and role/capability management at the first full-core access-management scope.

## Scope

- source-boundary inspection and implementation map;
- backend deterministic member status and role/capability capabilities;
- frontend typed member/role surfaces and action states;
- validation and verification.

## Acceptance criteria

- Disable/reactivate and role/capability preview/change have explicit capability ids, backend authorization, idempotency, no-op, last-admin/self-disable guardrails, audit/work traces, and safe denials.
- User Admin surfaces are attractive and workstream-native.
- Fullstack validation passes or bounded blocker tasks are appended.
