# Backlog 07: Foundation Workstream Expertise Expansion

## Outcome

The starter core app-description demonstrates that the workstream expertise model applies to all foundation functional agents, not only User Admin.

## Backlog items

1. **Audit foundation functional-agent expertise coverage**
   - Inventory `my-account-agent`, `user-admin-agent`, `agent-admin-agent`, `mission-control-agent`, `governance-policy-agent`, and `audit-trace-agent`.
   - Record which have detailed bundles, which need bundles, and which can be explicitly deferred or declared non-LLM/simple surface-only.

2. **Add Agent Admin expert bundle**
   - Cover governed AgentDefinition, prompts, skills, references, manifests, tool boundaries, behavior proposals, seed upgrades, approval paths, denials, traces, and tests.

3. **Add Audit/Trace expert bundle**
   - Cover trace search/explanation, redaction, sensitive evidence handling, export limits, denied load/tool traces, support-access audit, and tests.

4. **Add remaining foundation bundles or deferrals**
   - Add initial bundles or explicit readiness-impacting deferrals for Governance/Policy, Mission Control, and My Account.

5. **Update traceability/readiness and review**
   - Link new bundles to functional agents, capabilities, surfaces, observability, and tests.
   - Confirm every foundation functional agent has clear expertise status.
