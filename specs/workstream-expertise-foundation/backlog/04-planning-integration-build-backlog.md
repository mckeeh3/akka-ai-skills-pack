# Backlog 04: Planning and Queue Integration

## Outcome

Future plans, specs, backlogs, and pending queues generate bounded workstream expertise tasks for every new or materially changed functional agent.

## Backlog items

1. **Align PRD/spec/backlog guidance**
   - Require expertise bundle tasks when a PRD introduces or changes a functional workstream.
   - Preserve capabilities, authority, surfaces, prompt/skill/reference docs, manifests, traces, and tests in generated tasks.

2. **Align change-impact guidance**
   - Treat expertise changes as behavior, governance, UI, auth/security, observability, generation, and test changes.

3. **Align pending-task queue guidance**
   - Add bounded task families for workstream expertise.
   - Prevent vague `make the agent expert` or `agent governance` tasks from being marked runnable.

4. **Sprint review and next-task validation**
   - Generate or inspect an example plan for a hypothetical new domain workstream and record remaining gaps.
