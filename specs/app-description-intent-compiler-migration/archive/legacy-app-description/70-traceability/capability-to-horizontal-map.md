# Capability to Horizontal Map

Authoritative implementation candidates are maintained in `../60-generation/horizontal-implementation-map.md`.

- `secure-tenant-user-foundation` → Account/Profile/Settings KVE; Membership/Role component with audit-grade history where full-core requires it; Invitation ESE plus InvitationWorkflow; AdminAuditEvent write path; SupportAccessGrant component; `UserDirectoryView`, `MembershipView`, `InvitationView`, `AdminAuditView`, and selected `AccessReviewQueueView`; Resend email/outbox consumer; invite and support-access expiry timers; `/api/me`; `/api/admin/users/dashboard`, `/api/admin/users`, `/api/admin/users/{accountId}`, invitation/membership/role/account/support-access/access-review/audit HTTP APIs; Access/Profile, `user-admin-dashboard`, `user-admin-user-list`, and `user-admin-user-account` surfaces; optional governed UserAdminAgent read/summarize tools and governed Resend `@FunctionTool` preview/send surfaces for authorized agents. Dashboard/list/detail must be backed by scoped view queries (`UserDirectoryView`/`MembershipView`/`InvitationView`/`AdminAuditView`/`AccessReviewQueueView`) rather than caller-supplied known user ids; the first functional milestone may keep many mutations read-only/decision-card-only, but invitation resend/revoke or access-review decision-card creation must be wired as the initial safe mutation/approval path with AdminAuditEvent output.
- `ai-first-work-management` → goal/plan/decision entities, long-running plan workflows, planning/evaluation agents, governance-policy views, reminder timers, workstream/SSE APIs, Governance/Policy and Governance Workbench surfaces.
- `governance-decisions-audit` → policy/decision/audit entities, approval workflows, decision/audit views, trace-enrichment consumers, governance/audit agents, decision-card and audit-trace surfaces.
- `frontend-shell-integration-patterns` → HTTP endpoints/static hosting/SSE, React/Vite/TypeScript shell, functional-agent rail, workstream panel, composer, structured surface renderers, typed API client, UI tests.
- `managed-agent-foundation` → governed agent/document entities, proposal workflows, behavior editing/evaluator agents, catalog/history/trace views, protected HTTP APIs, `readSkill` capability, Agent Admin and Governance surfaces.
