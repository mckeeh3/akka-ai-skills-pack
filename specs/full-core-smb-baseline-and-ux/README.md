# Full-Core SMB Baseline and UX

## Purpose

Turn the umbrella SMB baseline and visual UX standard into executable shared contracts for the starter template before richer workstream implementation begins.

This child mini-project is repository-maintenance planning for the skills pack source. It prepares bounded implementation tasks for `templates/ai-first-saas-starter/`, synchronized `frontend/` assets when needed, docs, tests, and validation scripts.

## Source

Created from `specs/full-core-smb-saas-hardening/` during first-wave planning.

## Scope

- Shared workstream shell contract.
- Typed structured-surface and `system_message` contract.
- Visual quality acceptance map for dashboards and surfaces.
- Runtime validation map for local Akka/API/UI/provider-fail-closed behavior.
- Cross-workstream capability, trace, attention, and denial semantics.

## Non-goals

- Do not implement every workstream feature here.
- Do not create page-first CRUD navigation.
- Do not accept fixture-only or deterministic model-less normal runtime behavior for model-backed agent paths.

## Execution model

Execute one task per fresh harness session from `pending-tasks.md`.

## Done state

This child mini-project is complete when shared baseline/UX implementation contracts are ready for downstream workstream tasks, validated against the umbrella standards, and no missing shared-contract gap blocks User Admin full-core work.
