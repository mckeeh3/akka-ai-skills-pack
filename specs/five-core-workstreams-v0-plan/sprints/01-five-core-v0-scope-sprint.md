# Sprint 01: Five Core v0 Scope Coordination

## Objective

Define the shared v0 scope and dependency map that every five-core workstream mini-project must inherit.

## Scope

- shared workstream acceptance contract;
- agent-type selection rules across request/response AI, Akka `AutonomousAgent`, and deterministic non-AI services;
- dependency order and validation gates;
- consistency review of the five sibling queues.

## Acceptance criteria

- The plan identifies common runtime, auth, capability, trace, UI, and provider validation requirements.
- The workstream queues are aligned to the shared plan and can be executed independently.
- A terminal verification task can decide whether planning is complete or append follow-up tasks.
